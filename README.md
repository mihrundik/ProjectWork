# ProjectWork

# Инстукция по запуску тестов:

1. скачать на свой локальный компьютер проект
2. в пространстве проекта сформировать файл для установки переменных (пример во вложении - setenv.sh)
3. запустите терминал и перейдите в папку проекта
4. командой _source setenv.sh_ в терминале установить переменные
5. используя командную строку в терминале запустить тесты:
    - _mvn clean test_ - запуск с параметрами по-умолчанию
    - _mvn clean test -Dbrowser=safari_ - запуск с указанием браузера
    - _mvn clean test -Dbrowser=firefox -DfirefoxOptions='--kiosk,--incognito'_ - запуск с указанием браузера и опций
      через запятую
    - _mvn clean test -Dbrowser=chrome -DchromeOptions='--headless' -Dtest=NewWishlistDataTest_ - запуск с указанием браузера,
      опций и названия теста

---
1. Функциональность: выделено 5 критичных функциональностей и автоматизированы их тесты
Автоматизировано 15 тестов, которые покрывают 5 ключевых функциональных областей.
* Работа со списками желаний (вишлистами):
    * CreateNewWishlistTest.java: Успешное создание нового списка.
    * CloseFormTest.java: Закрытие формы создания через крестик.
    * CancelCreationTest.java: Отмена создания через кнопку "Отмена".
    * CreateWLLongName.java: Проверка валидации при очень длинном названии.
    * CreateWLLongDescription.java: Проверка валидации при очень длинном описании.
    * OpenCreateFormTest.java: Проверка открытия формы создания.
    * WishlistsExistTest.java: Проверка наличия списков.
* Навигация по спискам:
    * ClickButtonLastWLTest.java: Клик по кнопке "Просмотр" последнего списка.
    * WishlistNavigationTest.java: Проверка соответствия данных при переходе по кнопке "Просмотр".
    * NumberViewButtonsTest.java: Проверка количества кнопок "Просмотр".
    * ViewButtonsClickableTest.java: Проверка кликабельности всех кнопок "Просмотр".
* Удаление списков желаний:
    * VerifyWishlistDeletionTest.java: Проверка уменьшения количества списков после удаления.
    * DeleteLastWishlistTest.java: Удаление первого списка.
    * DeleteErrorMessageTest.java: Проверка сообщения об ошибке при попытке удалить список, который не может быть удален.
* Работа с подарками:
    * CreateNewGiftTest.java: Создание нового подарка в вишлисте.
    * AddGiftButtonTest.java: Проверка полей в окне создания подарка.
    * CancelAddGiftTest.java: Отмена создания подарка через крестик.
* Проверка данных:
    * NewWishlistDataTest.java: Проверка корректности данных нового вишлиста.
      
2. Технические требования
* Java: Проект собран на Java.
* Maven: Используется Maven для управления зависимостями (pom.xml).
* JUnit (Jupiter): Основной фреймворк тестирования.
* Логирование: Используется Log4j2 с SLF4J. Логи везде максимально детализированы.
* Page Object: Страницы (MyWishlistsPage, MyWishListPage, AddGiftPage) полностью изолируют логику работы с каждой отдельной сущностью. Тесты работают с абстрактными методами.
* Java Code Conventions: Код откомментирован, названия переменных и методов понятны.
* Кроссбраузерность: Реализована. Используется WebDriverManager и динамическое создание драйвера. Это позволяет запускать тесты в Chrome, Firefox, Safari и Edge.
* Удаленный запуск: Требование реализовано через использование переменных окружения и аргументов командной строки Maven.

3. Дополнительно
* Утилиты (utils): Классы WishlistHelper и WaitUtils — как пример переиспользования кода. Они призваны инкапсулируовать логику кода и делать тесты чище.
* Работа с переменными окружения (EnvConfig): Логин, пароль и URL тестового стенда вынесены в переменные окружения. Это соответствует лучшим практикам безопасности и гибкости.
* Обработка уникальных данных: Использование System.currentTimeMillis() для генерации уникальных названий списков и подарков предотвращает конфликты данных при многократном запуске.
